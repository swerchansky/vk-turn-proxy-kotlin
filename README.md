# VK-Turn-Proxy-Kotlin

Реализация идеи [vk-turn-proxy](https://github.com/cacggghp/vk-turn-proxy) на языке Kotlin с нативным Android-приложением. 

Выражаю огромную благодарность автору оригинального проекта за концепцию и идею. Данный проект адаптирует эту идею для удобного использования на мобильных устройствах.

Только для учебных целей взаимодействия траффика на Kotlin!!!

---

## Полная установка с нуля

### Шаг 1: Настройка WireGuard на сервере (VPS)
Самый простой способ — использовать автоматический скрипт установки:
1. Выполните команду на VPS:
   ```bash
   wget https://git.io/wireguard -O wireguard-install.sh && bash wireguard-install.sh
   ```
2. Следуйте инструкциям установщика. Запомните порт (по умолчанию `51820`).
3. Создайте клиента и скачайте полученный `.conf` файл или QR-код.

### Шаг 2: Настройка прокси-сервера на VPS
1. Скачайте бинарный файл сервера из [релизов GitHub](https://github.com/swerchansky/vk-turn-proxy-kotlin/releases):
   ```bash
   wget https://github.com/swerchansky/vk-turn-proxy-kotlin/releases/download/latest/<ваша платформа> -O server
   ```
2. Выдайте права на исполнение:
   ```bash
   chmod +x server
   ```
3. Запустите сервер в фоновом режиме:
   ```bash
   nohup ./server --listen 0.0.0.0:56000 --connect 127.0.0.1:51820 > server.log 2>&1 &
   ```
   *   `--listen 0.0.0.0:56000` — порт, который будет слушать прокси (его нужно будет указать в приложении).
   *   `--connect 127.0.0.1:51820` — адрес и порт вашего WireGuard сервера.
   *   Логи работы будут записываться в файл `server.log`.

### Шаг 3: Настройка Android-приложения
1. Установите APK из релизов.
2. Создайте ссылку на звонок.
3. В приложении:
    *   Вставьте **ссылку на звонок**.
    *   В поле **Server address** укажите `IP_вашего_сервера:56000`.
4. Нажмите кнопку **Connect**. Дождитесь статуса "Connected".

### Шаг 4: Настройка клиента WireGuard на телефоне
**Важно:** Прокси в приложении должен быть запущен *до* включения WireGuard.

1. Откройте приложение WireGuard и отредактируйте конфиг:
    *   **Endpoint**: измените на `127.0.0.1:9000` (адрес локального прокси).
    *   **MTU**: установите `1280` (важно для корректной работы через туннель).
2. **Исключения (Split Tunneling):**
    *   Зайдите в настройки туннеля -> "Исключённые приложения" (Excluded Apps).
    *   Найдите и отметьте приложение **vk-turn-proxy**. Это необходимо, чтобы само приложение имело доступ в интернет в обход VPN-канала.
3. Сохраните и включите туннель.

---

## Полезная информация
*   **MTU 1280** обязателен, так как инкапсуляция добавляет служебные заголовки.
*   Если соединение нестабильно, попробуйте изменить количество потоков (`-n`) в настройках приложения.
